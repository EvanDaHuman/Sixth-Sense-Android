================================================================================
SIXTH SENSE — OFFLINE KEYWORD TRIGGER
Instructions for an AI assistant setting this up on another machine
================================================================================

You (the assistant reading this) are helping a user run the keyword trigger that
lives in backend/normal_mode/. This document tells you exactly what it is, how it
works, what to install, how to run it, and how to extend it. Follow it top to
bottom. All commands assume macOS with zsh; Linux notes are included where they
differ.


--------------------------------------------------------------------------------
1. WHAT THIS IS
--------------------------------------------------------------------------------
A fully OFFLINE, terminal-only speech keyword detector for the Sixth Sense
project. It listens to the computer microphone, transcribes speech locally with
Vosk, and flags a "suspicious" reading whenever a hardcoded distress / boundary /
threat phrase is spoken.

Hard requirements that must NOT be violated:
  - NO cloud or paid transcription APIs (no Deepgram, OpenAI, Google, AWS).
  - Speech-to-text is Vosk, running locally/offline.
  - No website or browser page — it runs from the terminal.
  - No audio is recorded, stored, or uploaded. Only live transcript text is
    inspected in memory, then discarded.


--------------------------------------------------------------------------------
2. HOW IT WORKS (pipeline)
--------------------------------------------------------------------------------
  microphone
     |
     v
  sox / `rec`            -> captures mic as raw 16 kHz mono signed-16-bit PCM
     | (stdout, piped)
     v
  stt_vosk.py (Python)   -> feeds PCM into Vosk; prints final transcript lines
     | (stdout lines, piped)
     v
  monitorKeywords.js     -> reads transcript lines, calls detectKeywordTrigger()
     |
     v
  keywordTriggerEngine.js -> pure phrase matcher; returns matches + category

Why Python does the speech-to-text: the Node `vosk` package depends on
`ffi-napi`, which does NOT compile on current macOS toolchains (clang 21 / SDK 26
/ Node 23+). Python's Vosk ships prebuilt wheels, so it installs with no
compiler. Node still orchestrates everything and does the keyword matching, so
the command and the matching logic stay in JavaScript. Do not try to switch the
STT back to the Node `vosk` package — it will fail to build.


--------------------------------------------------------------------------------
3. FILES (all in backend/normal_mode/)
--------------------------------------------------------------------------------
  keywordTriggerEngine.js  Pure phrase matching. Exports:
                             - KEYWORDS: array of { phrase, category }
                             - detectKeywordTrigger(transcript) -> result object
                           No dependencies; runs on any Node version.
  monitorKeywords.js       The runner. Spawns sox (`rec`) and python3
                           stt_vosk.py, pipes mic audio in, reads transcripts
                           out, prints a block on each match, Ctrl+C to stop.
  stt_vosk.py              Vosk bridge: reads raw PCM on stdin, prints transcript
                           text on stdout. Resolves the model path (see below).
  package.json             Defines the `monitor-keywords` script. No Node deps.
  KEYWORDS.md              Human-readable version of these notes.
  .gitignore               Ignores model/, *.zip, __pycache__/, *.pyc.
  model/                   The Vosk model. **NOT in git** (too large). The user
                           must download it — see step 4.3. If it is missing the
                           runner prints MODEL_NOT_FOUND and download steps.


--------------------------------------------------------------------------------
4. SETUP (do these once, in order)
--------------------------------------------------------------------------------
4.1  SoX (microphone capture, provides the `rec` command)
       macOS:  brew install sox
       Linux:  sudo apt-get install -y sox        (also `alsa-utils` for a mic)
     Verify:   rec --version
     Note (macOS): if `rec` is "not found" when running, Homebrew may not be on
     PATH. Add it:  export PATH="/opt/homebrew/bin:$PATH"

4.2  Vosk for Python (prebuilt wheel — no compiling)
       python3 -m pip install vosk
     Verify:   python3 -c "import vosk; print('ok')"

4.3  Download an English Vosk model (~40 MB, NOT included in the repo)
       cd backend/normal_mode
       curl -LO https://alphacephei.com/vosk/models/vosk-model-small-en-us-0.15.zip
       unzip vosk-model-small-en-us-0.15.zip
       mv vosk-model-small-en-us-0.15 model
     The runner looks for, in order:
       1) $VOSK_MODEL_PATH  2) ./model  3) ./vosk-model-small-en-us-0.15
     (relative to backend/normal_mode/).

     There is NO `npm install` step — the Node side has zero dependencies.


--------------------------------------------------------------------------------
5. HOW TO RUN
--------------------------------------------------------------------------------
       cd backend/normal_mode
       npm run monitor-keywords

  (Equivalent to: node monitorKeywords.js)

  - First run, macOS asks for microphone permission for your terminal app —
    approve it (System Settings > Privacy & Security > Microphone).
  - It prints a banner, then "Listening on your microphone...".
  - Every non-matching utterance is echoed with a leading "·" so you can see it
    is hearing you.
  - Stop with Ctrl+C (it kills the sox and python subprocesses cleanly).


--------------------------------------------------------------------------------
6. EXPECTED OUTPUT
--------------------------------------------------------------------------------
  On a flagged phrase it prints exactly:

       KEYWORD TRIGGER DETECTED
       Mode: suspicious
       Matched phrase: <phrase>
       Category: <category>
       Transcript: <full transcript line>

  Example: saying "leave me alone" prints category "boundary"; "i will hurt you"
  prints category "threat"; "help" prints category "distress".


--------------------------------------------------------------------------------
7. FLAGGED PHRASES & CATEGORIES
--------------------------------------------------------------------------------
  Hardcoded in keywordTriggerEngine.js (KEYWORDS array). Categories:
     boundary : "leave me alone", "don't touch me"/"dont touch me", "get out"
     distress : "help", "i'm scared"/"im scared"
     threat   : "i'll hurt you"/"ill hurt you"/"i will hurt you",
                "i'm going to hurt you"/"im going to hurt you",
                "i'll kill you"/"ill kill you"/"i will kill you"

  Matching details:
   - Case-insensitive; apostrophes are stripped (so "don't" and "dont" both hit),
     other punctuation becomes spaces, whitespace is collapsed.
   - Overlapping variants collapse to ONE match (e.g. "i will hurt you" does not
     also fire "ill hurt you"). Genuinely different phrases in one utterance
     (e.g. "help" + "leave me alone") each produce their own block.

  To add/remove phrases: edit the KEYWORDS array in keywordTriggerEngine.js. Add
  both apostrophe and no-apostrophe spellings because offline STT often drops
  apostrophes.


--------------------------------------------------------------------------------
8. TESTING WITHOUT A LIVE MIC
--------------------------------------------------------------------------------
  Pure matcher (no audio, any Node version), from backend/normal_mode:
       node --input-type=module -e '
       import { detectKeywordTrigger } from "./keywordTriggerEngine.js";
       console.log(detectKeywordTrigger("please just leave me alone"));
       console.log(detectKeywordTrigger("the weather is nice"));
       '

  Full STT chain using synthesized speech (macOS `say`):
       say -o /tmp/kw.aiff "i will hurt you"
       sox /tmp/kw.aiff -q -r 16000 -c 1 -b 16 -e signed-integer -t raw - \
         | python3 stt_vosk.py


--------------------------------------------------------------------------------
9. INTEGRATING WITH THE REST OF SIXTH SENSE
--------------------------------------------------------------------------------
  detectKeywordTrigger(transcript) returns:
       {
         matched: boolean,
         mode: "normal" | "suspicious",
         phrase: string,        // first/primary match (when matched)
         category: string,      // "boundary" | "distress" | "threat"
         matches: [ { phrase, category }, ... ],
         transcript: string
       }
  Import it anywhere (it is dependency-free) to feed keyword hits into the
  project's suspicious-mode pipeline (e.g. backend/suspicious_mode/ or the
  servers). This keyword trigger is a NORMAL-MODE detector: it runs cheaply and
  locally, and only escalates by emitting a suspicious result on a match.


--------------------------------------------------------------------------------
10. TROUBLESHOOTING
--------------------------------------------------------------------------------
  "rec"/SoX not found        -> install sox (4.1); ensure /opt/homebrew/bin is on
                                PATH on macOS.
  python3 not found / STT
    exits immediately         -> install Python 3 and `python3 -m pip install vosk`.
  MODEL_NOT_FOUND             -> download the model (4.3) or set $VOSK_MODEL_PATH.
  No transcripts appear       -> grant microphone permission to the terminal app.
  Do NOT install the Node
    `vosk` package             -> it needs ffi-napi and will fail to compile on
                                modern toolchains; this design uses Python Vosk on
                                purpose.
